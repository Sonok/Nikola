# Daily Log — accountability

One file per day: `YYYY-MM-DD.md`. Fill it in as you go, not at the end of the night.

## Rules
1. Log every problem with: first-attempt result (solved / peeked / failed), time taken, and
   one sentence on the key idea IN YOUR OWN WORDS. If you can't write the sentence, you
   don't own the problem yet.
2. Every day starts with **yesterday's re-write** — hardest problem from yesterday, blank
   editor, before anything new. Log the time. This is the retention mechanism; skip it and
   the week evaporates.
3. Peeking at a solution is fine and expected (attempt 20 min → read → close → rewrite).
   Lying to the log about it is the only failure mode.
4. End each entry with tomorrow's plan while it's fresh.

## Template
```markdown
# YYYY-MM-DD (Day N of 10)

## Morning re-write
- [ ] <yesterday's hardest> — blank editor — time: ___ — clean? y/n

## Today's problems
- [ ] LC ___ <name> — result: solved/peeked/failed — time: ___
      key idea: <one sentence, your words>

## Non-LC prep (domain / api-design / resume-grill / behavioral)
- [ ] ...

## Honest note
<one line: what's shaky, what to hit again>

## Tomorrow
- [ ] ...
```
